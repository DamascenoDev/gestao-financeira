<objective>
Build the Settings surface: the RSC page (`/conta/configuracoes-ia`), the write-only `AiSettingsForm`, and the entry-point Card on `/conta` — mirroring the `mei/configuracoes` triad and the `delete-account-form` danger zone. The form mirrors `MeiSettingsForm` (manual state + `useTransition` + sonner, NOT react-hook-form) and the remove affordance mirrors `AccountDeleteZone`'s AlertDialog. All copy, states, and tokens come from 14-UI-SPEC.

Output: `src/app/(app)/conta/configuracoes-ia/page.tsx`, `src/components/ai-settings-form.tsx`, an entry Card on `src/app/(app)/conta/page.tsx`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| RSC → client (props/payload) | Only provider + has_key may cross; key_secret_id and the key must never be serialized into the RSC payload |
| key input (DOM) | The input is write-only — the stored key is never rendered as a value or masked placeholder |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-10 | Information Disclosure | Key/secret-id serialized into the RSC payload | mitigate | RSC selects key_secret_id only to compute hasKey and passes down the boolean; never a select('*'); verified in the Task 4 Network/RSC/bundle grep. (Pitfall 1) |
| T-14-11 | Information Disclosure | Stored key echoed into the form input | mitigate | Write-only input — no value from a stored key, no masked dots of the real key; help text + badge convey state instead. (Anti-Pattern 5) |
</threat_model>
