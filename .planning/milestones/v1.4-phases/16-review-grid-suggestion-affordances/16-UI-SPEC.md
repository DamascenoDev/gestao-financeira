## Color

The full token table below is the EXISTING v1.1 system (OKLCH in `globals.css`). This phase touches only the accent/secondary/consumption roles for the new affordances; it adds zero new colors.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` `oklch(0.985 0.004 255)` (light) / `oklch(0.2 0.03 258)` (dark) | Page + table surface |
| Secondary (30%) | `--secondary` `oklch(0.95 0.012 255)` / `--muted` | Cards, neutral pills — basis of the **"memória"** provenance badge |
| Accent (10%) | `--primary` (gold) `oklch(0.76 0.13 88)` / `oklch(0.82 0.13 88)` | Reserved for AI signal: the SuggestionSlot chip and the **"IA"** provenance badge |
| Destructive | `--destructive` `oklch(0.58 0.22 27)` | NOT used in this phase. Low confidence is NOT an error — never render it red. |

**Accent (gold `--primary`) reserved for, this phase only:**
1. The `SuggestionSlot` "Aplicar sugestão" chip (existing: `bg-primary/10 text-primary`, sparkles).
2. The **"IA"** provenance badge (gold-tinted + `Sparkles` icon) — visually echoes the chip so "this came from AI" reads as one family.

Accent is NOT used for: the "memória" badge (neutral), the confidence tag, the Select, or category dots.

**Confidence hint color:** `--consumption` `oklch(0.62 0.1 70)` (amber) — already the established "needs your attention / Classificar" hue in THIS grid (`text-consumption` placeholder at `import-review-table.tsx:754`, and `KindBadge` Consumo). Calm, non-alarming, never red. Used as `bg-consumption/10 text-consumption` for the "baixa confiança" tag.

**Provenance badge token map (mutually exclusive per row):**

| State | Condition | Token | Icon |
|-------|-----------|-------|------|
| memória | `category_id` set AND `classification_source === 'memória'` | `bg-secondary text-secondary-foreground` (neutral pill) | none |
| IA | `suggestion` present, `categoryId !== null`, AND `category_id === null` (unapplied) | `bg-primary/10 text-primary` (gold pill) | `Sparkles` size-3 |
| none | neither (manual, or `categoryId: null` "nenhuma encaixa", or no suggestion) | no badge rendered | — |

Color is never the sole signal: every badge/tag carries a text label (a11y), per the established `KindBadge` rule.

## Copywriting Contract

pt-BR, calm tone (matches existing grid copy).

| Element | Copy |
|---------|------|
| Primary CTA (apply suggestion) | **"Aplicar sugestão: {Categoria}"** (existing `SuggestionSlot` label — do not change) |
| Provenance badge — memória | **"memória"** (lowercase, neutral pill) |
| Provenance badge — IA | **"IA"** (gold pill + sparkles) |
| Confidence tag (low only) | **"baixa confiança"** (amber pill, shown only when `confidence < 0.6`) |
| Empty state heading | N/A — no new empty state. No key / no suggestions → grid is BYTE-IDENTICAL to v1.3 (no chips, no badges, no confidence tags, existing order). |
| Empty state body | N/A (see above) |
| Error state | N/A — this phase only READS `row.suggestion`; it makes no network/DB call and has no error surface. `categoryId: null` ("nenhuma encaixa") is a normal state: inert "—" slot, no chip, no badge. |
| Destructive confirmation | N/A — no destructive action. Applying a suggestion only fills the Select (NO commit); `confirmImport` is UNCHANGED and remains the sole `merchant_patterns` write, behind explicit human confirmation. |

**Confidence display rule (LOCKED):** the confidence hint is the "baixa confiança" TAG shown ONLY on low-confidence rows — chosen over an always-present tinted dot because it is calmer and keeps normal-confidence rows visually clean. It is NEVER a number or percentage. Threshold is a single tunable constant `LOW_CONFIDENCE = 0.6` (`confidence < 0.6` → show tag). Rows at/above threshold show no confidence affordance.

## Interaction & Layout Contract

The category cell at `import-review-table.tsx:744-771` is a `<div className="flex flex-col gap-1">`. The three affordances stack BELOW the existing `Select`, inside the slot the `SuggestionSlot` already reserves (`min-h-5`), so the Categoria column NEVER reflows whether or not a suggestion exists.

**Per-row rendering matrix (mutually exclusive top-level states):**

| Row state | Select | SuggestionSlot chip | Provenance badge | Confidence tag |
|-----------|--------|---------------------|------------------|----------------|
| memória (auto-classified, `category_id` set + `source 'memória'`) | shows `CategoryBadge` | none (inert "—" / nothing) | "memória" (neutral) | none |
| IA suggestion, `categoryId !== null`, not yet applied (`category_id === null`) | empty ("Classificar") | "Aplicar sugestão: {name}" | "IA" (gold + sparkles) | "baixa confiança" iff `confidence < 0.6` |
| IA suggestion applied (user clicked chip) | shows picked `CategoryBadge` | chip gone | optional "IA (aplicada)" if cheap, else nothing | none |
| "nenhuma encaixa" (`suggestion.categoryId === null`) | empty ("Classificar") | none (inert "—", as v1.3) | none | none |
| manual / no suggestion (v1.3 path) | as v1.3 | inert "—" (as v1.3) | none | none |

**Vertical order within the cell (top → bottom):** Select → [provenance badge] → SuggestionSlot chip (or inert "—") → [confidence tag]. Provenance + confidence sit on the chip row's reserved space; keep them on one wrap-friendly line (`flex flex-wrap items-center gap-1`) so a long category name + tag never force a column-width change.

**Apply mechanics (LOCKED):** clicking the chip calls the existing `SuggestionSlot.onApply(categoryId)` → fills the Select with the suggested category and marks origin as IA-applied. NO commit. `confirmImport` writes `merchant_patterns` via the SAME path as a manual pick — no pattern is written without explicit confirmation.

**Sort (LOCKED):** initial grid sort puts IA-suggested low-confidence rows FIRST (review the doubtful before the rest), then existing order. It is an INITIAL sort only — the user can re-sort by columns normally (`@tanstack/react-table`). Sort kicks in ONLY when AI suggestions exist in the upload; with no key / no suggestions the order is IDENTICAL to v1.3.

**Base UI guardrail (G-01):** the category `SelectValue` must keep explicit children (it already renders `CategoryBadge`); do not remove them when wiring the applied state.
