<objective>
Add the server contract for tagging a transaction to a carro (CAR-02): create/update
actions accept and persist an optional `carro_id` (with explicit clear-to-null), plus a
new `bulkTagCarro` action that tags many selected own transactions in one
`.in('id', ids)` update. Ownership of the target carro is re-derived via the Phase-8
tri-state `assertOwnedCarro` before every write (IDOR safety), and the write touches
ONLY `carro_id` — never category, amount, kind, the reserva ledger, or any metas
aggregate (D4, non-destructive lens).

This is the contract consumed by Plan 02 (extrato + transação-form UI) and Plan 03
(import-review UI), so it ships first as Wave 1.

Purpose: CAR-02's accounting-safe substrate — the single typed boundary every tagging
surface calls, with the D4 invariant and IDOR no-write proven by tests.
Output: `carroId` in the transaction schema; `carro_id` write/clear in
createTransactionWithReserva + updateTransaction; `bulkTagCarro`; action unit tests +
the integration D4/IDOR test (Wave 0).
</objective>
