<objective>
Add carro tagging to the import-review flow for CAR-02 (success-criteria #2): a per-row carro
selector in the pre-persist review grid that tags a parsed row to a carro BEFORE it is written,
and the confirmImport server path that persists `carro_id` from the per-row choice — re-deriving
carro ownership server-side and rejecting the whole payload on any forged carro_id (mirroring the
existing categoryId / reservaId re-derives). Tagging in review is non-destructive: it only sets
carro_id at insert; the category, amount, and reserva-aporte behavior are unchanged (D4).

Purpose: the user can attribute fatura line-items to a carro at the moment of review, without a
second pass on the extrato.
Output: optional carroId in the import row schema, the per-row review selector, the carro_id
persist path in confirmImport (ownership-checked), the review RSC carros wiring, and unit
coverage.
</objective>
