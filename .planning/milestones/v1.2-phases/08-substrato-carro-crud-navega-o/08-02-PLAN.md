<objective>
Build the Carro server layer: the Zod schema, the four CRUD/archive server actions, and the `assertOwnedCarro` ownership re-derive — the typed, validated, IDOR-safe boundary the Plan 03 UI calls. Mirrors the proven reservas/MEI action grammar exactly.

Purpose: CAR-01's write path. Every write must validate input with Zod, derive the owner from the session, re-derive ownership of any client-supplied carro id before touching it (FKs are not RLS-aware — the IDOR lesson), and return `{ ok: true } | { error }` without ever throwing. This is the contract the UI depends on, so it ships before the UI.

Output: `src/lib/schemas/carro.ts`, `src/actions/carros.ts`, `assertOwnedCarro` added to `src/lib/ownership.ts`, and `src/actions/carros.test.ts`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client form input → server action | Untrusted apelido/ano/combustivel/id cross here; validated by Zod + uuid guard |
| client-supplied carro id → FK / row write | A forged id must not let the caller mutate another user's carro |
| session claim → user_id | The owner of every insert derives from getClaims, never from client input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-06 | Elevation of Privilege / IDOR | updateCarro / archiveCarro / unarchiveCarro | mitigate | `assertOwnedCarro` re-derives ownership (exactly-1-row under RLS) before any write; forged id → { error } and NO update; asserted in carros.test.ts |
| T-08-07 | Tampering | carro field inputs (apelido/ano/combustivel) | mitigate | Zod `carroSchema` at the boundary (required apelido, ano bound, fixed combustivel enum); invalid input → { error }, never reaches the DB |
| T-08-08 | Spoofing | insert owner (user_id) | mitigate | user_id derived from `getClaims().sub`, never from client input; no session → { error: 'Sessão expirada.' } |
| T-08-09 | Denial of Service / leak | raw DB error surfaced to client | mitigate | DB errors mapped to friendly generic strings; raw details never returned; actions never throw (the { ok } | { error } contract) |
