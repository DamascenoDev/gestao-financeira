<objective>
Widen the `transactions.kind` CHECK constraint so PDF estornos/credits (D-04) can be persisted. Today `transactions.kind` is `text not null default 'expense' check (kind in ('expense'))` (migration 0005, line 16 — "expense only in P2"). The constraint only takes effect after `supabase db push`. The push needs `supabase link` + access token / DB password (interactive), matching the Phase 12 (12-02) precedent → `autonomous: false`.

Output: migration `0031_transactions_kind_credit.sql`; the constraint applied to the local stack AND the live production project (provisioned in Phase 12 / DEPLOY-01); regenerated `src/types/database.types.ts` with no drift.
