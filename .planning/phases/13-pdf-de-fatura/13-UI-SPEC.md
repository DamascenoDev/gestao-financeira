# Phase 13 — UI Design Contract

> Visual and interaction contract for the **PDF de Fatura** phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **SCOPE NOTE — additive delta, not greenfield.** The visual system (navy+gold identity, dark mode, spacing, typography, shadcn/base-ui component vocabulary) was locked in **Phase 7 (v1.1 "Identidade visual")** and is already shipped. The only genuinely new contracts are the five surfaces in **§New Surface Contracts**. Everything not listed there inherits the existing flow verbatim.

## Color

Locked navy+gold OKLCH system (`globals.css`). Re-stated; **no new color tokens this phase.** Approximate 60/30/10:

| Role | Value (token) | Usage |
|------|---------------|-------|
| Dominant (60%) | `--background` (navy-near-white light / deep-navy dark) | Page canvas |
| Secondary (30%) | `--card` / `--muted` / `--secondary` | Dropzone fill, grid rows, mobile cards, badges |
| Accent (10%) | `--primary` (muted metallic gold) | **Reserved-for list below** |
| Attention (calm) | `--consumption` (amber) | Unclassified / "needs your eyes" — a to-do, **never an error** |
| Destructive | `--destructive` (red) | Genuine failures + the **delete-row** action only |

**Accent (`--primary` gold) reserved for:** the primary CTA `Selecionar arquivo` button; the `Confirmar importação` button; the dropzone drag-over ring/fill (`ring-primary bg-primary/5`); gold focus ring (`--ring`); active nav. **Never** the unclassified signal, never the delete control, never row decoration.

**Color discipline carried into the PDF deltas:**
- A **gasto/expense is neutral** (`--foreground`/`--expense`), never red. The new **estorno/credit row** is rendered with the **`income` green token** (it reduces spend — same money-meaning rule as `amount-cell kind="income"`), NOT red, NOT gold.
- The **honest counters** stay calm: `descartadas` is `text-muted-foreground` (the system working, like `duplicadas`); only `não classificadas` escalates to amber. Partial PDF extraction is **expected**, so nothing in the PDF path turns red on partial coverage.
- The **delete-row** trigger is the *only* new use of `--destructive`, and only on the control affordance / its undo affordance — never painted across the row.

## New Surface Contracts

> These five are the *only* new/changed interaction surfaces. Each is expressed in the existing component vocabulary. Everything downstream of `ParseResult` (dedup, memory-classify, confirm, learn, route to `/extrato`) is unchanged.

### 1. PDF in the uploader (PDF-01)
- **`upload-dropzone.tsx`:** add `'pdf'` to `ACCEPTED_EXTS`; `<input accept=".ofx,.csv,.pdf">`; update `aria-label` to `Selecionar arquivo OFX, CSV ou PDF`. Wrong-type and size errors keep the existing inline `text-destructive role="alert"` treatment.
- **`import-uploader.tsx`:** extend `fileExt` return type to include `'pdf'`; no CSV-mapper pre-read for PDF (skip the header/sample branch — PDF has no column mapping).
- Helper copy + best-effort hint per the copywriting table. The hint sits in the existing `space-y-1` helper block under the CTA, muted `text-xs` — quiet, not alarming.
- Same direct-to-Storage signed-URL upload + `ingestStatement(path)` lifecycle; PDF just routes through the existing `UploadProgress` phases.

### 2. Delete-row in the review grid (D-02) — and NO add-row (D-03)
- **Placement:** a new trailing **`actions` column** in `import-review-table.tsx` (after `Valor`), header empty, `w-10`. On mobile (`<md`) the trigger sits in the card's bottom row, right-aligned next to the amount.
- **Control:** shadcn/base-ui `Button variant="ghost" size="icon"` with a lucide **`Trash2` `size-4`** icon; `text-muted-foreground hover:text-destructive`; `aria-label="Remover linha"`. Destructive intent shows only on hover/focus — the resting state is quiet so the grid does not read as a wall of red.
- **Behavior:** immediate client-state delete (mirror the `classifyRow`/`tagCarroRow` callback pattern): `setRows(prev => prev.filter(r => r.id !== id))`. Nothing is persisted; confirm sends only surviving rows. Fire a **sonner undo toast** (`Linha removida` / `Desfazer`) that re-inserts the row on undo. No `AlertDialog` modal (reserve modals for the existing K>0 "Importar sem classificar?" guard).
- **Keyboard / a11y:** the trigger is a real focusable `<button>`; `aria-label` present; reachable in tab order within the row; undo toast is announced by sonner. Color is never the sole signal (icon + label).
- **NO add-row control** (D-03). Do not add any "+"/"Adicionar linha" affordance anywhere in the grid.

### 3. Honest extraction counters (D-01) — and the post-delete bug fix
- **Reuse `import-summary-header.tsx`** for the layout. Add a distinct **`descartadas (J)`** count to the header (do NOT overload the existing `duplicadas` slot — descartadas and duplicadas are different concepts and may both be nonzero). Contract:
  - `total (N)` = rows extracted into the grid (initial parsed count).
  - `novas (M)`, `não classificadas (K)` = unchanged.
  - `descartadas (J)` = `ParseResult.dropped` (lines the extractor intentionally skipped: saldo/total/headers/`0,00`). This is **server-provided** and **fixed** — it must NOT change when the user deletes a grid row.
- **BUG TO FIX (RESEARCH-flagged):** today `ImportReviewTable` computes `duplicadas: Math.max(0, initialRows.length - rows.length)`. With delete-row, deleting a row shrinks `rows.length` and would mis-count user deletes as duplicates/descartadas. **Fix:** base the `descartadas`/`duplicadas` count on the **server-persisted summary value**, not the `initialRows.length - rows.length` delta. Track user deletes separately (they are not duplicates and not descartadas). The header value for `descartadas` reads from `summary` and stays stable across deletes.
- **Tone:** `descartadas` stays `text-muted-foreground` (calm — the system working). No red, no gate. Partial extraction never blocks; the grid is the safety net.

### 4. Image-only / zero-text empty state (PDF-04)
- **Hard block path** (`getText().text.trim().length === 0`): `ingestStatement` returns `{ error }`; the uploader shows the failure state. The review route must **never render a silent empty grid**. Use the existing `ui/empty` (`Empty` / `EmptyHeader` / `EmptyTitle` / `EmptyDescription` / `EmptyContent`) with the PDF-04 copy and a `Voltar para importar` button. This is the existing empty/error vocabulary — same as the "Importação não encontrada" and "Nada novo neste arquivo" states already on the review page.
- **Distinct from "0 rows but text present":** that case is NOT a block — render the review screen with honest `0 / {J} descartadas` counts + the `Nenhuma transação reconhecida` Empty body (copy table). The user sees the file was read.

### 5. Foreign-currency row (D-06, LOW confidence)
- **Graceful default only** — RESEARCH (A1) marks the layout unobserved in both samples. Do **not** build a heavy feature.
- Show the row like any other expense row; the `amount_cents` is the **BRL-converted value** (per D-06). Append a small muted `convertido` suffix tag to the descriptor (`text-xs text-muted-foreground`, inline, same grammar as the `descriptor_norm` line / `RecorrenteTag`). No second currency column, no rate display.
- The planner attaches a `checkpoint:human-verify` for the first real foreign-currency statement (per RESEARCH); the UI default must not break if the suffix data is absent.
