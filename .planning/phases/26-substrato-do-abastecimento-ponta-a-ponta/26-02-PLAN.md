## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated user → abastecimento_parcelas | New junction table; without RLS it is world-readable to any authenticated role (financial data leak). |
| view caller → consumo rows | `v_abastecimento_consumo` must run as the caller (security_invoker), never as its definer, or it bypasses every user's RLS. |
| direct insert / future action → abastecimentos | The relaxed CHECK is the DB-layer invariant that no write path can persist an invalid cost state. |
