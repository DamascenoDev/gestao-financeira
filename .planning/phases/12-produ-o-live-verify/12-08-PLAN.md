<root_cause>
`src/components/ui/select.tsx` re-exports `Select = SelectPrimitive.Root` (Base UI `@base-ui/react/select`). Base UI's `<Select.Value>` renders the selected `value` directly UNLESS one of two things is provided:

1. `Select.Root` is given an `items` prop — `Record<value, label>` (or `ReadonlyArray<{ value, label }>`). When set, `<Select.Value>` automatically renders the selected item's LABEL instead of the raw value. (Confirmed in `node_modules/@base-ui/react/select/root/SelectRoot.d.ts`: "When specified, `<Select.Value>` renders the label of the selected item instead of the raw value.")
2. `<Select.Value>` is given a children render-prop: `<SelectValue>{(value) => labels[value] ?? placeholder}</SelectValue>`. (Confirmed in `SelectValue.d.ts`: `children?: React.ReactNode | ((value) => React.ReactNode)`.)

The wrapper passes only `placeholder`, so every Select whose `value` ≠ visible label shows the raw value. The two grid tables (`extrato-table.tsx`, `import-review-table.tsx`) only look correct because they ALREADY pass a children render-prop to `SelectValue` that renders a `<CategoryBadge>`.

Standard preferred fix here: pass an `items` map to the `Select` (it forwards to `Root`) at each broken call site — this is the smallest, most idiomatic change and needs NO edit to `ui/select.tsx`. Use the render-prop form only where a map is awkward.
</root_cause>
