<root_cause>
`src/actions/incomes.ts` ALREADY exports `deleteOccurrence(id)` (validates the id with `idSchema`, getClaims owner, `.delete().eq('id', id)` under RLS, `revalidatePath('/receitas')`, returns `{ ok: true } | { error }`). The gap is purely UI: the receitas table (`/receitas/page.tsx`) renders only an `EditOccurrenceDialog` in the "Ações" cell — there is no delete trigger.

The recurring nuance: an occurrence with `template_id !== null` is a materialized instance of a template. Deleting it removes ONLY that month's row; `ensureMonthOccurrences` (materialize-on-read) will re-create it the next time the month is opened unless `ignoreDuplicates` finds it — i.e. deleting a recurring occurrence is inherently "só neste mês" and the month will re-materialize it on re-open. The confirm copy must state this so the user is not surprised. An avulsa (`template_id === null`) is removed outright.

Reusable pattern: `src/components/nf-table.tsx` → `NfRowActions` is the canonical Editar/Excluir DropdownMenu + AlertDialog confirm calling a delete server action (`deleteMeiInvoice`). Mirror it.
</root_cause>
