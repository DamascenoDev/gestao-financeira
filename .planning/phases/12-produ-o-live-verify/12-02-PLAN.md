<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local secret store (.env.local) → git | a committed .env.local would leak the project credentials |
| local migrations → remote project | a wrong db push could clobber pre-existing remote data |
| deployed bundle → public internet | the live client JS is world-readable; a secret here is fully exposed |
| service-role secret → client | SUPABASE_SECRET_KEY (LGPD delete) must stay server-only, never NEXT_PUBLIC_ |
| remote RLS / Storage | RLS + private bucket must hold on the REMOTE project, not just locally |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-envleak (I) | Information Disclosure | .env.local committed / secret in a NEXT_PUBLIC_ var / secret in Vercel public env | mitigate | .env.local gitignored; SUPABASE_SECRET_KEY set as a non-public server var; deployed-bundle grep gate (Task 1 + Task 4 devtools spot check, SEC-02) |
| T-12-clobber (T) | Tampering | db push overwriting pre-existing remote tables | mitigate | supabase db diff reviewed before push (D-12); migrations idempotent if-not-exists / create-or-replace |
| T-12-rls-remote (I) | Information Disclosure | RLS not actually enforced on the remote project | mitigate | dashboard check that NO table is "Unrestricted" (D-12) + live signup shows only the new user's own categories (Task 4) |
| T-12-storage-remote (I) | Information Disclosure | remote statements bucket public | mitigate | dashboard check that the bucket is PRIVATE before any upload flow runs (verified live in Plan 12-03) |
| T-12-authurl (S) | Spoofing / broken auth | Supabase Auth Site/Redirect URLs not pointing at the deployed URL → broken or hijackable redirects | mitigate | Task 4 sets Site URL + Redirect URLs to the *.vercel.app URL (D-06) |
| T-12-SC (T) | Tampering | npm/pip/cargo installs | accept | no new packages installed in Phase 12 (zero-dep phase); no Package Legitimacy Gate needed |
