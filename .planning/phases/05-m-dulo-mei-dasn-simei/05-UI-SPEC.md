# Phase 5 — UI Design Contract

> Visual and interaction contract for the **MEI / DASN-SIMEI module** — an independent sub-app under `(app)` that depends only on the Phase-1 foundation and does NOT touch the transactions/classification core. Scope: MEI dashboard / limite aplicável, Registrar NF (+ NF list), Configurações MEI, Relatório DASN-SIMEI (tela + export CSV + print), and the always-visible "informativo, não consultoria fiscal" disclaimer (MEI-06).
>
> **This phase INHERITS the app-wide design system established in Phase 2** (`02-UI-SPEC.md`) and **extended in Phase 3** (`03-UI-SPEC.md`): teal `--primary` accent (reserved), OKLCH semantic money tokens (`--income`/`--expense`/`--allocation`/`--consumption`/`--destructive`), `--font-mono` tabular numbers, dense tables, `base-nova` shadcn, pt-BR copy, money only through `src/lib/money.ts`, percentages via `Intl.NumberFormat('pt-BR', {style:'percent'})`, the Phase-3 `progress` bar + tiered/direction-aware status grammar, and the 80/100% alert affordance. This spec does **NOT** redefine that system — it extends it for the MEI screens. **Where this doc is silent, the Phase 2/3 contracts govern.**
>
> Generated by gsd-ui-researcher. Stack locked: Next.js App Router, TS strict, Tailwind v4 (`@theme`), shadcn `base-nova`, lucide, pt-BR. MEI limit/status math computed in SQL views/pure libs over the **gross** (bruta) revenue, never in the render path; the applicable limit is **always computed/passed in**, never a hardcoded `R$ 81.000` literal.

## Domain Constants (verified 2026 — `src/lib/mei.ts`)

> Confirmed against the current Receita rules during research (2026). Centralize; do not scatter literals. If a future year's figures change, this is the single edit site.

| Rule | Value (2026) | Cents | Notes |
|------|--------------|-------|-------|
| Teto anual cheio | R$ 81.000,00 | `8_100_000` | Full calendar-year ceiling. |
| Proporcional mensal | R$ 6.750,00 / mês ativo | `675_000` | 1º ano = `675_000 × mesesAtivos` where `mesesAtivos = 13 − mês(mei_start_date)` (mês de início inclusive até dezembro). |
| Banda de tolerância | 20% | — | Até R$ 97.200 (`9_720_000`) = migra para Simples no **ano seguinte**; acima de 20% = **desenquadramento retroativo** a janeiro do ano corrente. |
| Teto + banda | R$ 97.200,00 | `9_720_000` | `tetoCents × 1.2` derived, not stored separately. |
| Limiar de alerta | 80% do limite aplicável | — | Cruza `≥ 80%` → âmbar + alerta (MEI-05). |
| Prazo DASN-SIMEI | 31 de maio (ano seguinte) | — | Display-only reference in the relatório; not a blocking date. |

**Applicable-limit rule (MEI-02, the subtle part):**
- **1º ano** (calendar year of `mei_start_date`) → **proporcional**: `675_000 × (13 − startMonth)` cents. Ex.: início em abril → 9 meses → R$ 60.750.
- **Anos seguintes** → **limite cheio** R$ 81.000.
- The UI label for the limit is *computed* — e.g. "Limite proporcional 2026 · R$ 60.750,00 (9 meses a partir de abril)" or "Limite anual 2027 · R$ 81.000,00". **Never** a bare "R$ 81.000" string.

## Semantic Color — MEI Limit Status (no new tokens)

The status is computed by `meiStatus(grossCents, limitCents)` and maps onto **existing** tokens, reusing the Phase-3 tiered grammar exactly. The 20%-band sub-distinction is carried by **label + glyph**, not a new color.

| Condition (`gross ÷ applicable limit`) | Status | Fill / text token | Glyph | Label |
|----------------------------------------|--------|-------------------|-------|-------|
| `< 80%` | **verde** (tranquilo) | `--income` (green) | `check-circle-2` | "Dentro do limite" |
| `≥ 80%` e `< 100%` | **âmbar** (aproximando) | `--consumption` (amber) | `triangle-alert` | "Aproximando do limite" |
| `≥ 100%` e `≤ banda` (`≤ R$ 97.200`) | **vermelho** (dentro da banda de 20%) | `--destructive` (red) | `alert-octagon` | "Acima do limite — dentro da tolerância de 20%" |
| `> banda` (`> R$ 97.200`) | **vermelho** (fora da banda) | `--destructive` (red) | `octagon-x` | "Acima da tolerância de 20% — risco de desenquadramento" |

**Rules carried over (respected):**
- Green (`--income`) means "good outcome" = comfortably under the limit — consistent with income = money in.
- Red (`--destructive`) appears **only** at over-limit (`≥100%`), exactly as Phase 3 reserves red for an estourado `teto`. Both red states use `--destructive`; the **within-band vs over-band** difference is conveyed by the **status label + glyph + a one-line consequence sentence**, never by inventing a second red.
- Color is never the sole signal: every state pairs the token with a text status label AND a lucide glyph (survives grayscale / color-blindness), and the gauge exposes `aria-valuetext`.
- Neutral chrome stays grayscale; color appears only on the gauge fill, the % figure, the status badge, and the alert glyph — never on card backgrounds.

`@theme` already exposes `bg-income`/`bg-consumption`/`bg-destructive` — no `globals.css` change. **No color token work this phase.**

## Disclaimer (MEI-06) — placement & treatment

The disclaimer is a **requirement and a liability concern**, so it is **visible, not a tiny footer**.

- **Placement:** rendered **once** in the `/mei` route-segment layout, as a **persistent banner directly under each MEI page's h1** (above the screen's primary content), so it is on-screen on every MEI screen (dashboard, NF, settings, report) without scrolling.
- **Treatment:** a full-width **`--muted` surface** rounded notice with `md` padding, an `info` lucide glyph (not an alarm/destructive glyph — this is informative, calm), and `text-sm` body in `text-muted-foreground` with the key clause in `text-foreground` weight 600. It is **not dismissible** (no close button) — it must remain present.
- **Not accent, not destructive:** the banner does NOT use teal (reserved for actions) nor red (reserved for over-limit). It is deliberately neutral so it reads as a standing notice, not an error.
- **Print:** the disclaimer is **included** in the print/PDF view of the relatório (it must survive export-to-print), placed in the report header.
- **Copy:** **"Este módulo é informativo e não constitui consultoria fiscal."** + a second line: "Os valores e o status são uma estimativa baseada nos seus registros. Confirme suas obrigações com um contador ou no Portal do Empreendedor."

### 1. MEI dashboard / limite aplicável (`(app)/mei`) — headline screen

The headline: where the user sees gross-revenue-vs-applicable-limit and the tiered status at a glance.

- **Header:** h1 "MEI" + `YearSelector` (`?ano=`). `MeiDisclaimer` banner directly below (from the segment layout).
- **Limite card (`--card`):**
  - **Hero:** "Receita bruta em {ano}" label (`text-xs muted`) over the accumulated gross figure (**28px mono/600**, colored by current status token).
  - **Limite aplicável line:** computed label + value (e.g. "Limite proporcional 2026 · R$ 60.750,00 — 9 meses a partir de abril/2026", or "Limite anual 2027 · R$ 81.000,00"). **Always computed, never a hardcoded 81k** (MEI-02). Mono value.
  - **`LimiteGauge`:** track + tiered fill + 80% tick + 100% marker (+ band marker when relevant). The % figure (`receita bruta ÷ limite`) in mono, status color, beside the gauge.
  - **`LimiteStatusBadge`:** the verde/âmbar/vermelho(within-band)/vermelho(over-band) badge with glyph + label.
  - **Remaining-to-limit line** (when under 100%): "Faltam R$ X para o limite" (`text-muted-foreground`, mono).
- **Alert affordance (MEI-05):** at `≥ 80%` the card surfaces an **inline amber alert row** under the gauge: "Você atingiu {%} do seu limite de {ano}. Acompanhe seus próximos registros." (no toast/notification system in v1 — the dashboard IS the alert surface, mirroring Phase 3). At `≥ 100%` within band: a red consequence line "Acima do limite. Dentro da tolerância de 20% você migra para o Simples Nacional no ano seguinte." At `> band`: "Acima da tolerância de 20% — risco de desenquadramento retroativo a janeiro de {ano}. Procure um contador."
- **Quick links:** secondary buttons/links to "Registrar NF", "Relatório DASN-SIMEI", "Configurações" (the latter especially when no `mei_start_date` is set — see states).
- **States:**
  - **Empty (no `mei_start_date` configured):** `empty` — heading "Configure seu MEI para começar" · body "Informe a data de início do seu MEI para calcularmos o limite aplicável e acompanhar seu faturamento." · CTA "Configurar MEI" (→ `/mei/configuracoes`). The limit cannot be computed without a start date — surface this as copy, never a wrong/zero limit.
  - **Empty (configured, no NFs in {ano}):** limite card shows R$ 0,00 (verde, 0%) and a sibling `empty`/hint "Nenhuma nota fiscal registrada em {ano}. Registre sua primeira NF." · CTA "Registrar NF".
  - **Near-limit (≥80%):** amber gauge + alert row (above).
  - **Over-limit / over-band (≥100% / >20%):** red gauge + the corresponding consequence line + glyph.
  - **Loading:** `skeleton` for hero + gauge (keep header + disclaimer visible).
  - **Error:** inline `text-destructive` "Não foi possível carregar os dados do MEI. Tente recarregar a página."

### 4. Relatório DASN-SIMEI (`(app)/mei/relatorio`) — annual consolidated report

The consolidated, print-friendly, exportable annual report with *exactly* the DASN-SIMEI fields (MEI-04).

- **Header:** h1 "Relatório DASN-SIMEI" + `YearSelector` + secondary CTA **"Exportar CSV"** (`ExportCsvButton`) + a print affordance ("Imprimir" / browser print, print-friendly layout). `MeiDisclaimer` below h1 **and** included in the print header.
- **`DasnReportView` (print-friendly card):**
  - **Cabeçalho:** "Ano-base {ano}" + período coberto (jan–dez {ano}) + DASN deadline reference ("Prazo de entrega: 31 de maio de {ano+1}").
  - **Receita bruta total:** hero mono figure (the same accumulated gross as the dashboard).
  - **Split por atividade:** "Comércio, indústria e transporte intermunicipal/interestadual" total + "Prestação de serviços" total — two mono figures that **sum to the total** (a tiny reconciliation note if rounding ever diverges, though integer cents won't). This is the exact DASN split.
  - **Empregado durante o ano-calendário:** **Sim / Não** (from `has_employee` for `{ano}`).
  - Layout: a clean two-column label/value grid (`--card`), generous `lg` section gaps, mono right-aligned values. No teal, no decorative color — this reads like a document.
- **Print layout:** a print stylesheet (or print-targeted classes) hides the sidebar/nav, keeps the disclaimer + report card, and renders on white. Money never wraps.
- **CSV export:** rows = the report fields (receita bruta total, total comércio/indústria, total serviços, funcionário sim/não, ano, limite aplicável). pt-BR-safe (UTF-8 BOM, `;` or `,` delimiter decided in the export helper). Establishes the reusable CSV pattern (Phase 6 DATA-01). Toast "Relatório exportado".
- **States:**
  - **Empty (no NFs in {ano}):** still renders the report with zeros + a note "Nenhuma receita registrada em {ano}." — the report is valid (a MEI with no revenue still declares); do not block.
  - **Empty (no `mei_start_date`):** info row linking to Configurações (the applicable-limit reference cannot be computed without it), but the DASN totals (which don't need the limit) still render.
  - **Loading:** `skeleton` for the report card.
  - **Error:** inline `text-destructive` "Não foi possível gerar o relatório. Tente recarregar a página."

## Money & Number Formatting

Inherited from Phase 2/3, with MEI specifics:

- **Money:** every value via `formatCents` (pt-BR `R$ 1.234,56`); every input via `parseBRLToCents` (throws → `FieldError` "Valor monetário inválido."). Mono + `tabular-nums`. NF values and report totals are **receita BRUTA**.
- **Percentages:** mono, `tabular-nums`, `maximumFractionDigits:1` → `74,2%`, `100%`, `120%`. The limit % may read **>100%**; never `NaN%` — if the applicable limit is unknown (no start date), show the "Configure seu MEI" state, not a division.
- **Dates:** `dd/MM/yyyy` display; year boundaries via `date-fns-tz` pinned to `America/Sao_Paulo` (`startOfYear`..`endOfYear`). The 1º-ano proportional month count derives from `mei_start_date`'s month in SP.
- **Limit math:** computed by `src/lib/mei.ts` pure helpers (and/or a SQL view, `security_invoker=true`) over the gross sum — the UI only formats the output. **No fiscal literal lives in a component.**

## Accessibility Basics

Inherited from Phase 2/3, plus:

- **Color never sole signal:** every MEI status pairs color with a text status label ("Dentro do limite"/"Aproximando do limite"/"Acima do limite — dentro da tolerância"/"Acima da tolerância de 20%") and a lucide glyph — survives grayscale and color-blindness. The within-band vs over-band difference is **textual**, never color-only.
- **`LimiteGauge`:** `role="progressbar"` + `aria-valuenow`/`aria-valuemin`(0)/`aria-valuemax`(limite cents) + `aria-valuetext` ("Receita bruta R$ X de R$ Y — 74,2% do limite, aproximando").
- **`YearSelector`:** arrows `aria-label` "Ano anterior"/"Próximo ano"; current year announced.
- **`MeiDisclaimer`:** rendered as a standing region (e.g. `role="note"`), persistently in the accessibility tree on every MEI screen.
- **Forms:** `field` primitive (label association + `aria-invalid` + `FieldError`) mirroring `auth-form.tsx`; the `activity_type` and `has_employee` controls have accessible names.
- **Destructive (delete NF):** `alert-dialog`, focus-trapped, escape-cancellable.
- **CSV/print:** the "Exportar CSV" and print buttons have accessible names; print view preserves the disclaimer in the document order.
- **Contrast:** status tokens tuned ≥4.5:1 for text, ≥3:1 for the gauge fill / badge non-text indicators, light and dark (tokens already tuned in Phase 2).
