<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Human walkthrough — MEI module end-to-end in the browser</name>
  <what-built>
    The complete MEI module (Plans 01–03), running against the LOCAL stack: schema + summary view, IDOR-safe NF/settings actions, the dashboard (LimiteGauge + status badge + alert), the NF form/list, the settings form, the DASN report + CSV export, and the persistent disclaimer.
  </what-built>
  <action>
    Pause for the user to walk the MEI screens in the browser and confirm the behaviors below. Do not proceed until the user types "approved" or reports issues.
  </action>
  <how-to-verify>
    Open the app (dev server) and sign in as your local test user, then:

    1. Configurações (MEI-03): visit `/mei/configuracoes`. Set a MEI start date in the CURRENT year but NOT January (e.g. a date in abril) so the first-year cap is proportional. Toggle "Tinha funcionário em {ano}?" on, save. Confirm the toast "Configurações salvas".
    2. Dashboard proportional limit (MEI-02): visit `/mei`. Confirm the limit line reads a COMPUTED proportional value (e.g. "Limite proporcional {ano} · R$ 60.750,00 — N meses a partir de {mês}/{ano}") and NOT a bare "R$ 81.000". With no NFs yet the gauge is verde at 0%.
    3. Registrar NF (MEI-01/03): visit `/mei/notas`, click "Registrar NF". Add a few NFs across the year, mixing activity types (Comércio/Indústria and Serviços), with values that together reach roughly 60–85% of the proportional limit. Confirm each saves (toast), appears in the list with the right Atividade badge + green `+` value, and the year-total footer matches.
    4. Gauge + status + alert (MEI-02/05): return to `/mei`. Confirm the hero gross matches the year total, the gauge fill + % reflect gross ÷ applicable limit, the status badge reads the right tier, and — once you cross 80% — the amber "Você atingiu {%}…" alert row appears. (Optionally add more NFs to push over 100% and confirm the red within-band consequence line.)
    5. Edit/delete an NF: edit one NF's value and delete another; confirm the list, year total, and dashboard gauge all update.
    6. Relatório DASN-SIMEI (MEI-04): visit `/mei/relatorio`. Confirm "Receita bruta total" equals the year gross, the Comércio + Serviços split sums to the total, "Empregado durante o ano-calendário" reads Sim (from step 1), and the DASN deadline line reads "31 de maio de {ano+1}". Click "Exportar CSV" and open the downloaded `dasn-{ano}.csv` — confirm it contains the same total, split, employee flag, and that pt-BR amounts render correctly (open in a spreadsheet to confirm the BOM/`;` make columns split). Use the print affordance and confirm the print view keeps the disclaimer + report on white without the sidebar.
    7. Disclaimer (MEI-06): on EACH of /mei, /mei/notas, /mei/configuracoes, /mei/relatorio confirm the "Este módulo é informativo e não constitui consultoria fiscal." banner is visible above the content (not a tiny footer) and is calm/neutral (not teal, not red).
    8. Year isolation: change the YearSelector to a different year and confirm the NFs/limit/report reflect THAT year (the previous year's NFs do not bleed in).
  </how-to-verify>
  <resume-signal>Type "approved" if all eight checks pass, or describe any issues (which screen, what you expected vs saw) so they can be fed into a `/gsd-plan-phase 5 --gaps` replan.</resume-signal>
