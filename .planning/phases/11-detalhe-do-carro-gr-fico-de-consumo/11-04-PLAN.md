<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
    The enriched `/carros/[id]` detail page (header + 3 KPI cards: km/l médio · R$/km · gasto total + gasto-por-categoria bars + km/l-over-time consumption line chart + the integrated Phase-10 abastecimento history) and the `/carros` list cards now carrying gasto total + km/l médio. All values pt-BR `R$`, mono `tabular-nums`, `—` for no-data. Chart line and bar fill are token-aware (flip with dark mode). recharts SVG / computed colors / theme flip / tooltip / responsive layout are NOT measurable in jsdom — this is the only thing left to verify by eye.
  </what-built>
  <how-to-verify>
    Start the app (`npm run dev`) and sign in to the LOCAL stack. Seed (or use) a carro that has: at least 2 full-tank abastecimentos forming valid km/l intervals, and a few expenses tagged to it across 2+ categories.

    1. Go to `/carros`. Each card shows "Gasto total" + "km/l médio" under the identity line, mono numerals, neutral (not gold/red). A car with no spend/abastecimento shows `—` (NOT "R$ 0,00", NOT "0 km/l"). Identity link + Editar/Arquivar dropdown still work.
    2. Open `/carros/[id]` for the seeded car. Confirm section order top→bottom: header (apelido + modelo·placa·ano) → 3 KPI cards (km/l médio, R$/km, Gasto total) → "Gasto por categoria" bars → "Consumo (km/l)" chart → "Abastecimentos" history.
    3. KPI cards: the 3 figures are mono `tabular-nums`; a car missing a value shows `—`; gasto total is neutral foreground (NOT red).
    4. Gasto por categoria: one bar per categoria, ordered largest→smallest, each with the categoria name + a mono `R$` amount; the widest bar is the largest spend; fill is a NEUTRAL grey tint (NOT gold, NOT green/amber money colors).
    5. Consumo chart: a gold line (the `--chart-1` series) over time; hover a point → tooltip reads "12,4 km/l" style pt-BR; the curve has NO point for intervals without a valid km/l (no gaps drawn as 0). A car with <2 valid intervals shows "Sem dados de consumo ainda" instead of a chart.
    6. Theme flip: toggle dark mode. The chart line, bar fills, KPI text, and card surfaces all re-theme correctly — no invisible line, no broken contrast, money/neutral semantics preserved in BOTH light and dark.
    7. Mobile: shrink to a phone width (or DevTools device mode). KPI cards stack to one column; the chart stays full width and readable; the Phase-10 abastecimento table collapses to cards (Phase-7 grammar); nothing overflows.
    8. Empty: open a carro with no tagged spend → "Nenhum gasto vinculado a este carro." under "Gasto por categoria" (no zero bars); with no valid intervals → the chart empty copy.
  </how-to-verify>
  <resume-signal>Type "approved" if all eight checks pass in light AND dark on desktop AND mobile; otherwise list the visual issues (which check, what's wrong) and they become a gap-closure plan.</resume-signal>
