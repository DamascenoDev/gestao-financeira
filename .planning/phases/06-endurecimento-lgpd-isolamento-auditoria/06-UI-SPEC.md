# Phase 6 ŌĆö UI Design Contract

> Visual and interaction contract for the **Endurecimento (LGPD, isolamento, auditoria)** phase. Most of this phase is backend + tests (2-user isolation harness, secret/Storage/PII audits); the UI surface is **small and deliberately careful**: a new **Privacidade / Conta** screen under `(app)` (LGPD data export + the most dangerous action in the app, account deletion) and a **transactions CSV export** affordance reusing the Phase-5 `ExportCsvButton` pattern. Covers DATA-01, DATA-02 (UI parts); SEC-01 has no UI.
>
> **This phase INHERITS the app-wide design system established in Phase 2** (`02-UI-SPEC.md`) and extended through Phases 3ŌĆō5: teal `--primary` accent (reserved), OKLCH semantic money tokens (`--income`/`--expense`/`--allocation`/`--consumption`/`--destructive`), `--font-mono` tabular numbers, dense tables, `base-nova` shadcn, pt-BR copy, money only through `src/lib/money.ts` (`formatCents`/`parseBRLToCents`), the **Phase-2 destructive `alert-dialog` grammar** (`AlertDialogTitle` + `AlertDialogDescription` "Esta a├¦├Żo n├Żo pode ser desfeita." + `[Cancelar]` / destructive `[Excluir]`), and the **Phase-5 CSV-export pattern** (`ExportCsvButton` Blob download + a pure `*ToCsv` serializer with `String.fromCharCode(0xFEFF)` BOM + `;` delimiter + pt-BR money). This spec does **NOT** redefine that system ŌĆö it extends it for the two new surfaces. **Where this doc is silent, the Phase 2ŌĆō5 contracts govern.**
>
> Generated by gsd-ui-researcher. Stack locked: Next.js App Router, TS strict, Tailwind v4 (`@theme`), shadcn `base-nova`, lucide, pt-BR. The account-delete Server Action is **server-only / service-role** (DATA-02): the service-role key is the first legitimate use in the project and MUST stay off the client bundle (the secret-grep audit proves it). The UI never imports it ŌĆö it only calls the action and reacts to the result.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (already initialized ŌĆö `components.json` present, `registries: {}`) |
| Preset | `base-nova` (inherited) |
| Component library | Radix via base-nova primitives (vendored) |
| Icon library | lucide |
| Font | `--font-sans` (Geist) for prose; `--font-mono` (Geist Mono) for any counts shown (e.g. "{n} transa├¦├Ąes") |

**Design intent (this phase).** This is a *safety* surface, not a feature surface. Two opposing tones live on one screen:
1. **Export is calm and reassuring** ŌĆö a neutral `--card` section that tells the user exactly what they get and lets them download it. No alarm color.
2. **Delete is loud and frictional on purpose** ŌĆö the most dangerous action in the app. It earns the full weight of `--destructive`, an unmissable consequences list, and a type-to-confirm gate. The friction is the feature: it must be hard to do by accident and impossible to do by reflex.

The teal accent does NOT appear on this screen's dangerous half ŌĆö destructive is red, export is a calm secondary/outline. Teal stays reserved (active nav item only here).

  - "Todos os dados do MEI (notas, configura├¦├Ąes e relat├│rios)"
  - "Os padr├Ąes de classifica├¦├Żo que o sistema aprendeu"

## Money & Number Formatting

Inherited from Phase 2/5, with this phase's specifics:

- **Transactions CSV (`transactionsToCsv`):** money via `formatCents` (pt-BR `R$ 1.234,56`, decimal comma); BOM via `String.fromCharCode(0xFEFF)`; `;` delimiter; `\r\n` line endings; dates `dd/MM/yyyy`. Sign/value derived from `kind` (never from a negative number) ŌĆö consistent with the Extrato's `AmountCell`. A category-less or odd row still serializes a valid line. **Mirror `src/lib/mei/csv.ts` exactly** and unit-test parity.
- **LGPD bundle:** the CSVs inside the bundle reuse the same pt-BR-safe serialization (BOM + `;` + `formatCents`); the JSON carries raw integer-cents (the machine-readable form) so the export is both human-openable (CSV) and faithful (JSON cents). No money is hand-formatted.
- **Counts:** any "{n} transa├¦├Ąes" shown renders mono, `tabular-nums`.
- **Dates:** display `dd/MM/yyyy`; file-name dates `yyyy-MM-dd` (sortable); month window for the transactions CSV pinned to `America/Sao_Paulo` via the existing `?mes` helper.

## Accessibility Basics

Inherited from Phase 2ŌĆō5, plus the dangerous-dialog specifics:

- **Danger dialog focus order:** `AccountDeleteDialog` is focus-trapped; **initial focus is on `Cancelar`**, not the destructive confirm ŌĆö a reflexive Enter cancels, never deletes. Escape cancels (never confirms).
- **Type-to-confirm:** the confirm button is `disabled` (+ `aria-disabled="true"`) until the typed phrase matches `APAGAR` exactly; the input has an accessible label and its helper ("Para confirmar, digite APAGAR") is linked via `aria-describedby`. The disabledŌåÆenabled transition is not color-only ŌĆö the button label/state and the typed phrase carry it.
- **Consequences are in the a11y tree:** the irreversibility list lives in `AlertDialogDescription` (or is referenced by it), so a screen reader announces what will be deleted before the user reaches the confirm input. It is **never** a hover-only tooltip.
- **Color never sole signal:** the danger zone reads from its heading text + `triangle-alert` glyph + the explicit list, not red alone; the export section reads as calm/neutral by absence of alarm color, with explicit copy.
- **Export buttons:** `ExportDataButton` and `ExportTransactionsButton` have accessible names ("Baixar meus dados", "Exportar transa├¦├Ąes em CSV"); their busy state ("PreparandoŌĆ”"/"ExportandoŌĆ”") is announced via the disabled button text (and `sonner` is a polite live region).
- **Nav:** the new "Conta" sidebar item and the `UserMenu` "Privacidade e conta" item have visible focus rings (`--ring`/`--primary`) and accessible names.
- **Contrast:** `--destructive` text/border tuned Ōēź4.5:1 (text) / Ōēź3:1 (border indicator) against its surface in light and dark (token already tuned in Phase 2). The mono **APAGAR** word meets text contrast.
