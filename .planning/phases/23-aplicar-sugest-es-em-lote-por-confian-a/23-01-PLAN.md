<objective>
Refine the review-grid bulk-apply action (CLSAI-10): instead of applying ALL unapplied AI
suggestions in one click, apply ONLY suggestions at/above the existing `LOW_CONFIDENCE = 0.6`
threshold (per D — "Semântica do aplicar-em-lote"). Low-confidence (`< 0.6`) rows stay
untouched and uncategorized → pending for per-row manual review. Nothing is committed: the
fill is client-state only (`origin → 'manual'`, `reserva_id` null), and `confirmImport` (the
sole `merchant_patterns`/`transactions` write path) stays byte-identical.

This is a single-file behavior edit confined to `import-review-table.tsx` plus its test.
No server/schema/types/migration/package change — the schema-push gate stays OFF.

Purpose: Make the bulk action trustworthy — only confident guesses are auto-filled; the
operator reviews uncertain ones line-by-line. The threshold is made explicit in the LOCKED
pt-BR copy ("confiáveis"). Per D ("Limiar"), reuse the existing `LOW_CONFIDENCE` constant so
the "confiável" set and the existing amber "baixa confiança" set partition the space at 0.6
with no gap/overlap and no second source of truth.

Output: 4 micro-edits to `import-review-table.tsx` + the updated/extended component test.
</objective>
