## Copywriting Contract

pt-BR, calm tone, singular/plural mirrored from today's code. **These two strings are the entire user-visible deliverable of this phase — they are LOCKED here.**

| Element | Copy |
|---------|------|
| Primary CTA (bulk-apply button label) | **"Aplicar {N} sugestões confiáveis"** — plural; singular form **"Aplicar 1 sugestão confiável"** when `N === 1`. `{N}` = the confident pending count (`confidentSuggestionCount`), NOT the old `unappliedSuggestionCount`. The word "confiáveis/confiável" makes the threshold explicit per CLSAI-10 / CONTEXT.md. |
| Success toast | **"{N} sugestões confiáveis aplicadas"** — plural; singular form **"1 sugestão confiável aplicada"** when `N === 1`. Replaces today's "{N} sugestões aplicadas" (`import-review-table.tsx:409-411`). |
| Button visibility (not copy, but locked here) | Render the Button ONLY when `confidentSuggestionCount >= 1`. Hide it when the confident count is 0, **even if low-confidence rows still remain pending** — those go to per-row manual review. Mirrors today's hide-when-none (`unappliedSuggestionCount > 0 ? ... : null`, `import-review-table.tsx:719`). |
| Empty state | N/A — no new empty state. When `confidentSuggestionCount === 0` the Button simply isn't rendered; the grid is otherwise byte-identical to today (pending low-confidence rows keep their amber tag + first-sort). |
| Error state | N/A — bulk-apply only mutates client grid state (fills the Select; `origin → 'manual'`, `reserva_id` null). No network/DB call, no error surface. `confirmImport` is UNCHANGED and stays the sole `merchant_patterns`/`transactions` write behind explicit human confirmation. |
| Destructive confirmation | N/A — no destructive action, no undo. Nothing is committed; the applied rows remain editable/overwritable per-row (CONTEXT.md: "calmo, sem undo"). |

**Plural rule (LOCKED, matches existing code style):** use the same `N === 1 ? singular : plural` ternary already present at `import-review-table.tsx:410` and `:727`. Do not introduce an i18n library or a different pluralization helper. The count is a plain integer, never a percentage.

## Interaction & Layout Contract

**Single point of change:** `src/components/import-review-table.tsx`. No other production file, no server route, no schema/types.

**Count source (LOCKED semantics, copy fixed above):** the Button's count and visibility are driven by a new **confident pending** count — rows that are still unclassified AND carry a confident AI suggestion:
`category_id === null && suggestion?.categoryId != null && suggestion.confidence >= LOW_CONFIDENCE`.
This REPLACES the old `unappliedSuggestionCount` (`import-review-table.tsx:707-709`) as the driver of the Button. `LOW_CONFIDENCE = 0.6` (`import-review-table.tsx:109`) is reused as the single threshold — no new constant. The exact helper name/shape (e.g. `confidentSuggestionCount`) and whether the confident predicate is factored to share the threshold with the existing low-confidence predicate is **Claude's discretion** per CONTEXT.md.

**Apply mechanics (LOCKED):** `applyAllSuggestions` (`import-review-table.tsx:393-415`) is gated to apply ONLY confident rows — same per-row gate plus `&& r.suggestion.confidence >= LOW_CONFIDENCE`. Rows below threshold are left **intocadas e sem categoria** (untouched, no category) → they stay pending for per-row manual review and keep their existing amber "baixa confiança" tag + first-sort position. The fill is client-state only (`origin → 'manual'`, `reserva_id` null), NEVER a DB write. `confirmImport` stays the sole merchant_patterns/transactions path — UNCHANGED.

**Boundary semantics (LOCKED):** `confidence >= LOW_CONFIDENCE` → applied (confiável); `confidence < LOW_CONFIDENCE` → stays pending — consistent with the existing "baixa confiança" tag predicate (`< 0.6`, `import-review-table.tsx:181`).

**Feedback (LOCKED):** one success toast with the applied confident count (copy above). No spinner, no undo, no confirm dialog — the action is non-binding and reversible by per-row re-edit.

**Must stay byte-identical (do NOT touch):** the gold `Sparkles` icon (`size-4`), the Button `variant="outline"` and its header placement/gap, the "Confirmar importação" primary Button, the amber "baixa confiança" tag, the low-confidence-first initial sort, per-row `OriginBadge`/`SuggestionSlot`, and every memória/keyword binding path (those arrive pre-filled with `category_id` + origin and carry nothing "pending" — the bulk-apply count never includes them).
