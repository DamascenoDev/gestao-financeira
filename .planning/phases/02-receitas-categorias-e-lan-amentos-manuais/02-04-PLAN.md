| bulk action → transactions UPDATE | Forged ids must not touch other users' rows |
| RSC → v_category_totals | Totals must run under the caller's RLS |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-02-TXN-BULK | Tampering | bulkReclassify .in('id', ids) | mitigate | RLS scopes the UPDATE to the caller's rows even with forged ids; bulk-reclassify.test.ts asserts user B's forged-id update touches 0 rows |
| T-02-TXN-VAL | Tampering | createTransaction / updateTransaction | mitigate | Zod schema (categoryId uuid, occurredOn date regex, amount string) + parseBRLToCents throw-on-invalid; amount_cents > 0 DB check |
| T-02-TXN-RLS | Information Disclosure | transaction reads + v_category_totals | mitigate | RLS on transactions (02-01) + security_invoker v_category_totals; transactions-rls.test.ts proves four-verb isolation |
</threat_model>
