0004_incomes.sql — income_templates (id, user_id FK auth.users ON DELETE CASCADE, source text, amount_cents bigint check >= 0, day_of_month int check between 1 and 31, is_active boolean default true, created_at) + income_occurrences (id, user_id, template_id uuid references income_templates ON DELETE SET NULL [null = avulsa, INC-03], source text snapshot, amount_cents bigint check >= 0, month_key text check matching the YYYY-MM regex, occurred_on date, created_at, unique (user_id, template_id, month_key)). Index income_templates_user_id_idx and income_occurrences_user_month_idx on (user_id, month_key). RLS + grants on both. The unique constraint makes materialize-on-read idempotent (INC-02) and — because Postgres treats NULLs as distinct — still allows multiple avulsas per month (INC-03).

0005_transactions.sql — transactions (id, user_id, category_id uuid references public.categories ON DELETE RESTRICT [blocks hard-delete, CAT-02], amount_cents bigint check > 0 [ALWAYS positive; sign derives from kind, not value], kind text default 'expense' check (kind in ('expense')), occurred_on date, description text default '', created_at). Indexes transactions_user_month_idx on (user_id, occurred_on) and transactions_category_idx on (category_id). RLS + grants.

0006_categories_color.sql — additive only: alter table public.categories add column if not exists color text (swatch key slate|red|amber|green|teal|blue|violet|pink per UI-SPEC; nullable, no data migration). Do NOT touch existing columns or the seed trigger.

0007_views.sql — v_income_month with (security_invoker = true) = select user_id, month_key, sum(amount_cents)::bigint as total_cents from income_occurrences group by user_id, month_key. v_category_totals with (security_invoker = true) = select user_id, to_char(occurred_on,'YYYY-MM') as month_key, category_id, sum(amount_cents)::bigint as total_cents, count(*)::int as tx_count from transactions group by user_id, to_char(occurred_on,'YYYY-MM'), category_id. grant select on both to authenticated. security_invoker = true is MANDATORY (PG 17 local) — without it the view runs as definer and leaks every user's sums (T-02-VIEW).

0008_reassign_and_delete.sql — reassign_and_delete_category(src uuid, dst uuid) as a "security invoker" plpgsql function doing UPDATE transactions set category_id = dst where category_id = src then DELETE from categories where id = src, in one transaction (atomic move-then-delete, RESEARCH Open Question 2). grant execute to authenticated.
