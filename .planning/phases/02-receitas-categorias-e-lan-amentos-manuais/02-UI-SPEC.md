# Phase 2 вҖ” UI Design Contract

> Visual and interaction contract for the first substantial UI in the app. Phase 1 shipped only auth forms + a placeholder dashboard, so **this spec establishes the app-wide design system** (palette, type scale, density, semantic tokens) that Phases 3 (dashboard/reservas) and 5 (MEI) inherit. Scope: app shell + Receitas + Categorias + TransaГ§Гөes/Extrato.
>
> Generated by gsd-ui-researcher, verified by gsd-ui-checker. Stack is locked: Next.js App Router, TS strict, Tailwind v4 (`@theme`), shadcn/ui `base-nova`, lucide, pt-BR. Money formatting/parsing goes **only** through `src/lib/money.ts` (`formatCents`, `parseBRLToCents`).

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (already initialized вҖ” `components.json` present) |
| Preset | `base-nova` (style), `neutral` baseColor, CSS variables, RSC |
| Component library | Radix (via base-nova primitives already vendored) |
| Icon library | lucide |
| Font | `--font-sans` (existing Geist sans via `next/font`); `--font-mono` (Geist Mono) reserved for numeric/money columns |

**Design intent (non-generic direction).** The product is a private, trustworthy money ledger seen daily. The visual language is *calm, dense, and quietly precise* вҖ” not a colorful consumer budgeting app. Rules that make it feel intentional rather than templated:

1. **Color is information, never decoration.** The neutral grayscale chrome stays grayscale. Saturated color appears **only** to carry financial meaning (income vs expense, consumo vs alocaГ§ГЈo, category identity, over/under). A screen with no money semantics on it should read as near-monochrome.
2. **Numbers are first-class.** All money and counts render in `--font-mono` with tabular figures and right-alignment in tables so columns scan vertically. Money is the loudest typographic element on the Extrato, louder than descriptions.
3. **Density over whitespace on the Extrato.** The central screen is a tight, readable table (compact row height), not airy cards. Cards/airiness are reserved for forms and empty states.
4. **One accent, reserved.** A single calm teal primary carries the brand + primary CTA. It is never used to mean "income" or "category" вҖ” those have their own dedicated tokens вҖ” so the accent never competes with financial signal.

## New Design Tokens (app-wide вҖ” add to `globals.css`)

The existing `:root`/`.dark` palette is fully grayscale (primary/accent/chart all neutral). This phase **introduces the real palette as additive OKLCH CSS variables** under `@theme inline` + `:root`/`.dark`, so every later phase inherits them. Keep all existing shadcn tokens; add these.

| Token | Light (OKLCH) | Dark (OKLCH) | Meaning |
|-------|---------------|--------------|---------|
| `--primary` (override) | `0.55 0.07 195` | `0.72 0.07 195` | Calm teal вҖ” brand, primary CTA, active nav. Trust, not alarm. |
| `--primary-foreground` (override) | `0.985 0 0` | `0.205 0 0` | Text on primary |
| `--income` | `0.62 0.13 155` | `0.72 0.14 155` | Receita / money in (green). |
| `--income-foreground` | `0.985 0 0` | `0.205 0 0` | Text on income fill |
| `--expense` | `0.205 0 0` | `0.985 0 0` | Gasto / money out вҖ” **neutral foreground, not red**. Expense is normal, not an error. |
| `--allocation` | `0.58 0.10 240` | `0.70 0.11 240` | AlocaГ§ГЈo (investimento/poupanГ§a) вҖ” indigo. Distinguishes "saving" from "spending". |
| `--consumption` | `0.55 0.07 60` | `0.70 0.09 60` | Consumo (gasto) вҖ” warm amber. |
| `--destructive` (existing) | `0.577 0.245 27.325` | `0.704 0.191 22.216` | Errors + destructive confirmations + over-limit only. |

Mirror each new token into `@theme inline` (e.g. `--color-income: var(--income)`) so Tailwind utilities `bg-income`, `text-income`, `border-allocation` exist.

**Category colors** (the optional per-category color, CAT decision): a fixed 8-swatch palette the user picks from (no free hex picker in v1 вҖ” keeps the extrato legible and avoids clashing with semantic tokens). Store the chosen swatch key (`slate|red|amber|green|teal|blue|violet|pink`) on the category; render as a small dot/badge. Swatches are mid-chroma OKLCH (вүҲ`L 0.6 / C 0.12`) tuned to read on both card and table surfaces.

**Hard rule for money color:** sign/effect derives from `kind`, never from a negative value (matches the data model вҖ” `amount_cents` is always positive). Render an income occurrence with `text-income` and a leading `+`; render a gasto in neutral `text-foreground` with a leading `вҲ’` only in mixed contexts. Never paint a gasto red unless it breaks a limit (Phase 3 concern).

## Spacing Scale

8-point scale (multiples of 4). Maps to Tailwind defaults already in use.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, badge padding, dot-to-text |
| sm | 8px | Compact spacing, table cell vertical padding (dense rows) |
| md | 16px | Default element spacing, form field gaps, card padding |
| lg | 24px | Section padding, dialog padding, page header gap |
| xl | 32px | Layout gaps between major blocks |
| 2xl | 48px | Empty-state vertical centering |
| 3xl | 64px | Reserved (not needed this phase) |

**Exceptions:**
- Dense Extrato table rows target **40px** row height (`py-2` + text line) for scan density вҖ” a deliberate sub-48 touch target on desktop. On mobile the Extrato switches to a card-per-transaction layout (see Responsive) where tap targets are вүҘ44px.
- Interactive icon-only controls (row checkbox, row actions menu) get a **44px** hit area via padding even when the visible glyph is smaller.

## Typography

Three sizes for prose + one mono numeric role. Two weights only (400 / 600).

| Role | Size | Weight | Line Height | Notes |
|------|------|--------|-------------|-------|
| Body / table cell | 14px (`text-sm`) | 400 | 1.5 | Extrato cells, list items, descriptions. 14 (not 16) because this is a dense data tool. |
| Label / meta | 12px (`text-xs`) | 400 | 1.4 | Table headers (uppercase, `tracking-wide`), badges, helper text, `text-muted-foreground` |
| Heading | 20px (`text-xl`) | 600 | 1.2 | Page title (one per screen), dialog title |
| Money / numeric | 14px mono | 600 | 1.5 | `--font-mono`, `tabular-nums`, right-aligned in tables. Totals row uses same size, weight 600. The "receita lГӯquida do mГӘs" hero figure may step up to 28px mono/600. |

Rules: exactly one `text-xl` h1 per page. No bold-on-bold. Weight 600 is reserved for headings, money, and active nav вҖ” not for general emphasis (use `text-foreground` vs `text-muted-foreground` for hierarchy instead).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` (white / near-black) | Page background, main content surface, Extrato table body |
| Secondary (30%) | `--card` / `--muted` / `--sidebar` | Top nav bar, side nav, cards, dialogs, table header row, selected-row highlight (`--muted`) |

**Accent (teal `--primary`) reserved for:** primary action button (one per surface), active navigation item, focus rings, and the checked state of selection controls. **Never** used for income (green owns that), never for category color, never as a generic link color (links use `--foreground` underline). This keeps the 10% honest вҖ” on the Extrato the only teal is the "Novo lanГ§amento" button and any active filter.

60/30/10 holds: most pixels are neutral background/text; nav + cards + table chrome are the secondary neutral surfaces; teal + the financial semantic colors together stay under ~10% of surface and only where they carry meaning.

## Screen Contracts

### 0. App Shell / Navigation (`(app)/layout.tsx`)

Replaces the current minimal header (`GestГЈo Financeira` + logout). Establishes app chrome for all later phases.

- **Layout:** persistent left **sidebar** (collapsible to icons on `md`, off-canvas drawer on mobile) + top bar. Sidebar nav items (lucide icon + label): Dashboard (`layout-dashboard`, placeholder), Receitas (`arrow-down-circle`), Categorias (`tags`), Extrato/TransaГ§Гөes (`receipt-text`). Active item uses `--primary` text + subtle `--muted` background (base-nova `menuAccent: subtle`).
- **Top bar (rightвҶ’left):** `MonthSelector` (center-left of content), then user dropdown (`dropdown-menu`) containing the email + Sair (logout, AUTH-04 вҖ” keep existing `LogoutButton` action inside the menu).
- **Month context is global:** the selected month drives Receitas and Extrato. Default current month. Stored in URL searchparam, read in server components.
- **States:** no empty/loading state for the shell itself; it renders instantly around streamed page content.
- **Theme:** respect light/dark via existing `.dark` class. No theme toggle required this phase (system default acceptable); tokens must be defined for both.

### 1. Receitas (`(app)/receitas`)

Register/edit recurring income templates + ad-hoc incomes; per-month occurrence edit; show "receita lГӯquida do mГӘs".

- **Header:** h1 "Receitas" + `MonthSelector` already in shell. Hero strip: **"Receita lГӯquida do mГӘs"** label (`text-xs muted`) over the summed figure (28px mono/600, `text-income`) вҖ” this is the Phase 3 denominator, make it prominent.
- **Body:** a `table` (or two via `tabs`: Recorrentes / Avulsas) listing this month's occurrences. Columns: Fonte (text) В· Tipo (badge: Recorrente/Avulsa) В· Valor (`AmountCell`, income green `+`) В· aГ§Гөes.
- **Recurring template editing distinction (INC-02):** editing a row's value opens a dialog asking *"Alterar sГі em {mГӘs}"* (edits the materialized occurrence) **vs** *"Alterar o template"* (future months). This choice must be explicit copy, never silent.
- **Create:** primary CTA "Nova receita" вҶ’ `dialog` with `field` form: Fonte (text-free, `Select`/combobox with suggestions SalГЎrio/PensГЈo/Outros), Valor (`MoneyInput`), tipo recorrente toggle (`switch`) вҖ” if recorrente, show day-of-month; if avulsa, show a date.
- **Empty state** (`empty`): heading "Nenhuma receita neste mГӘs" В· body "Cadastre seu salГЎrio ou um recebimento avulso para comeГ§ar a calcular suas metas." В· CTA "Nova receita".
- **Loading:** `skeleton` rows. **Error:** inline `text-destructive` "NГЈo foi possГӯvel carregar as receitas. Tente recarregar."

### 2. Categorias (`(app)/categorias`)

Manage list; create/rename; consumo/alocaГ§ГЈo toggle; optional color; archive/reassign on delete-with-transactions.

- **Body:** `table`/list. Columns: cor (swatch) + nome В· Tipo (`switch` consumoвҶ”alocaГ§ГЈo, inline editable, CAT-03) В· usos (txn count, muted) В· aГ§Гөes.
- **Create/rename:** "Nova categoria" CTA вҶ’ `dialog`: nome (`input`), tipo (`switch` consumo/alocaГ§ГЈo, default consumo), cor (8-swatch picker). Seed categories editable but flagged.
- **Special-category warning:** when editing a seed category consumed by a feature (e.g. **Reserva**, used by Phase 3 reservas), show an inline `text-muted-foreground` warning under the field: "Esta categoria Г© usada pelo fluxo de reservas вҖ” alterГЎ-la pode afetar relatГіrios."
- **Delete-with-transactions (blocked hard-delete):** the delete action opens an `alert-dialog`. If the category has transactions, **do not offer destructive delete**; instead offer two paths: **Arquivar** (hide from pickers, keep history) or **Reatribuir e remover** (a `select` to pick a target category вҶ’ moves transactions, then deletes). Copy: "Esta categoria tem {n} transaГ§Гөes. VocГӘ nГЈo pode excluГӯ-la diretamente." + the two buttons.
- **Empty state:** unlikely (seed exists) but handle: "Nenhuma categoria" В· "Crie categorias para organizar seus gastos." В· CTA.

### 3. TransaГ§Гөes / Extrato (`(app)/extrato`) вҖ” central screen

Dense readable table; month + category(multi) filters; quick add/edit; per-category totals; multi-select bulk re-classify (reused by Phase 4 memory flow вҖ” selection UX matters).

- **Layout priority:** density + fast read. Default = current month (from shell). This is the screen the user sees most.
- **Filter bar** (above table): month is the shell `MonthSelector`; **category multi-select** (`popover` with checkbox list or a multi-`select`) showing active filters as removable `badge`s; a quick "Limpar filtros" link. Filters live in URL searchparams.
- **Table** (`@tanstack/react-table` + shadcn `table`): columns вҖ” `checkbox` (select) В· Data (`dd/MM`) В· DescriГ§ГЈo (truncate + `tooltip`) В· Categoria (`CategoryBadge`, inline-editable via a small `select` on click) В· Valor (`AmountCell`, mono right-aligned). Sort by data desc default. Row height ~40px, zebra-free, `--border` dividers.
- **Per-category totals:** a sticky footer (or a collapsible summary strip above) showing each filtered category and its summed `formatCents` total, plus a grand total. Mono, right-aligned, weight 600.
- **Quick add:** primary CTA "Novo lanГ§amento" вҶ’ `dialog` form (data `popover`+`calendar`, descriГ§ГЈo, valor `MoneyInput`, categoria `select`). Also support **inline category edit** directly in the row (click the badge вҶ’ select) for fast re-tagging without opening a dialog.
- **Bulk re-classify (TXN-04) вҖ” the load-bearing selection UX:**
  - Header checkbox selects all *filtered* rows; per-row checkboxes select individually; shift-click selects a range.
  - When вүҘ1 row is selected, a **selection action bar** appears (sticky, `--card` surface): "{n} selecionadas" В· a category `select` В· primary button "Reclassificar" В· "Limpar seleГ§ГЈo". Applying shows a `sonner` success toast "{n} transaГ§Гөes reclassificadas" with an **Desfazer** action where feasible.
  - Selection state and the action bar are designed to be reused verbatim by the Phase 4 import-review/memory flow вҖ” keep it a self-contained component (`SelectionActionBar`).
- **States:**
  - **Empty (no transactions this month):** `empty` вҖ” heading "Nenhuma transaГ§ГЈo em {mГӘs}" В· body "Lance um gasto manualmente para comeГ§ar seu extrato." В· CTA "Novo lanГ§amento".
  - **Empty (filters too narrow):** "Nenhuma transaГ§ГЈo com esses filtros" В· CTA "Limpar filtros".
  - **Loading:** `skeleton` table rows (keep header + filter bar visible).
  - **Dense data:** virtualize or paginate only if needed; default render all rows for a month (bounded). Totals always reflect the current filter, not the page.
  - **Error:** inline `text-destructive` row "NГЈo foi possГӯvel carregar o extrato." + retry.

## Money & Number Formatting

- **Display:** every money value rendered via `formatCents(cents)` (pt-BR `R$ 1.234,56`). Never hand-format. Mono + `tabular-nums` in tables.
- **Input:** every money entry parsed via `parseBRLToCents` at submit; the helper throws on invalid вҶ’ surface as a `FieldError` ("Valor monetГЎrio invГЎlido."). Show `R$` affix.
- **Sign/color:** derived from `kind` (income вҶ’ green `+`; gasto вҶ’ neutral, `вҲ’` only in mixed lists). Never from a negative number.
- **Dates:** `dd/MM/yyyy` display, `dd/MM` in dense table. Month label "Junho 2026" (date-fns pt-BR). All month boundaries via `date-fns-tz` pinned to `America/Sao_Paulo` вҖ” a late-night transaction must not slip months.
- **Counts:** integers, mono in totals.

## Responsive Behavior

Desktop-first (primary use), usable on mobile (responsive web, no native вҖ” per PROJECT scope).

- **Sidebar:** full labels вүҘ`lg`; icon-rail `md`вҖ“`lg`; off-canvas drawer (`sidebar` mobile mode) `<md`.
- **Extrato `<md`:** table collapses to **one card per transaction** (DescriГ§ГЈo + Categoria badge on top, Data + Valor on a second line). Selection becomes a tap-to-select with a floating bottom selection bar. Per-category totals move to a collapsible summary at the top.
- **Filters `<md`:** collapse into a single "Filtros" `popover`/sheet trigger.
- **Dialogs `<md`:** full-width sheet-style; forms stack single-column (they already are).
- Money/number cells never wrap; truncate descriptions with tooltip instead.

## Accessibility Basics

- All interactive controls keyboard-reachable; visible focus ring uses `--ring`/`--primary` (already in base layer `outline-ring/50`).
- Form fields use the `field` primitive (label association + `aria-invalid` + `FieldError`) вҖ” mirror `auth-form.tsx`.
- Selection checkboxes have accessible names ("Selecionar transaГ§ГЈo {descriГ§ГЈo}", header "Selecionar todas").
- Color is never the sole signal: kind shows a text badge (Consumo/AlocaГ§ГЈo) alongside color; income shows `+` and a label, not only green; category shows name beside the dot.
- `MonthSelector` arrows have `aria-label` ("MГӘs anterior"/"PrГіximo mГӘs"); current month announced.
- Toasts (`sonner`) are polite live regions; destructive confirmations use `alert-dialog` (focus-trapped, escape-cancellable).
- Contrast: semantic tokens tuned to вүҘ4.5:1 against their surface for text, вүҘ3:1 for the dot/badge non-text indicators, in both light and dark.
