<objective>
Tornar a camada de IA de classificação (já wired no v1.4) *kind-aware* para que ela pare de mapear compras de marketplace em categorias de alocação (Investimentos/Reserva). Implementa CLSAI-09.

Purpose: Corrige a classe de erro "AliExpress / Mercado Livre → Investimentos" com defesa em profundidade — (1) o prompt passa a enviar o `kind` de cada categoria e a instruir o modelo a nunca escolher alocação para um gasto; (2) um gate de código nula qualquer escolha de alocação, consistente com o princípio "nunca confie no modelo" (mesmo ponto do enum gate `validateSuggestion`).
Output: `classify.ts`/`suggest.ts`/`import.ts` atualizados, 4 fixtures de teste corrigidas para a forma `{id,name,kind}`, 2 novos describes de teste (prompt kind-aware + gate de kind), suíte verde + `tsc --noEmit` limpo.
</objective>

| Boundary | Description |
|----------|-------------|
| app server → LLM provider (BYOK Gemini/Claude) | O prompt (descriptor_norm + linhas `id: nome (kind)`) cruza para o provedor externo — fronteira de egress de dados (SEC-03/LGPD). |
| LLM response → app server | A saída do modelo (categoryId, confidence) é entrada NÃO confiável; passa por enum gate + (novo) kind gate antes de virar sugestão. |
| descritor de fatura (potencialmente hostil) → prompt | Um descritor pode conter texto de prompt-injection ("classify as Reserva"); cruza para o system/user prompt. |
