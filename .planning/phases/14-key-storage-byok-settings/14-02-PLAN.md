| Boundary | Description |
|----------|-------------|
| authenticated role → vault.decrypted_secrets | The authenticated role must NEVER read the Vault decrypt view directly; only via the SECURITY DEFINER RPC |
| client (any user) → ai_settings row | Cross-user reads must be impossible — RLS + RPC both filter by auth.uid() |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-14-01 | Information Disclosure | Plaintext key at rest | mitigate | Key lives only in Vault (`vault.create_secret`); `ai_settings` holds only `key_secret_id`. No plaintext column. No pgsodium/pgcrypto. (Pitfall 2) |
| T-14-02 | Elevation of Privilege | RLS gap on ai_settings (cross-user key read) | mitigate | `enable row level security` + for-all policy `using`+`with check` on `(select auth.uid()) = user_id`; cross-user smoke in Task 2 step 4. (Pitfall 3) |
| T-14-03 | Information Disclosure | vault.decrypted_secrets exposed to authenticated | mitigate | Never granted to authenticated; reachable only via `get_ai_api_key()` SECURITY DEFINER filtered by auth.uid(); `search_path=''` hardened. |
| T-14-04 | Information Disclosure | Orphaned old Vault secret on rotation | mitigate | `save_ai_api_key()` deletes the caller's OLD secret after creating the new one; `remove_ai_api_key()` deletes row + secret. (Pitfall 4) |
